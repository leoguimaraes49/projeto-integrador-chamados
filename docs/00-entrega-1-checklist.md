# Entrega 1 - Checklist

## Base extraída dos documentos da disciplina

Pelos PDFs analisados, a disciplina espera que o projeto evolua em ciclos. Para a primeira entrega, os pontos mais importantes são:

- Aula 1: grupos formados, tema/proposta de projeto definido e registro inicial no GitHub Projects.
- Aula 2: Documento de Visão e Product Backlog inicial.
- Avaliação: o Documento de Visão vale 10% e deve apresentar escopo, stakeholders e proposta de valor.

## Escopo desta entrega

Projeto escolhido: sistema web para gerenciamento de chamados de suporte técnico.

Stack definida para esta base:

- Node.js e Express no backend.
- PostgreSQL como banco de dados relacional.
- JWT para autenticação.

## O que registrar no GitHub Projects

Criar um board com os seguintes campos mínimos:

- Status: Backlog, Ready, In Progress, In Review, Done.
- Prioridade: Alta, Media, Baixa.
- MoSCoW: Must, Should, Could, Won't.
- Tipo: Feature, Bug, Tech, Docs.
- Sprint: Sprint 1, Sprint 2, Sprint 3.

Itens sugeridos para inserir inicialmente:

- PB-01 a PB-08 como prioridade da Sprint 1.
- PB-09 a PB-15 como candidatos da Sprint 2.
- PB-16 a PB-20 como candidatos da Sprint 3 ou melhorias futuras.
