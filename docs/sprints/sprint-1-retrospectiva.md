# Sprint 1 - Retrospectiva

## Start

- Dividir as tarefas de API, testes e documentacao em partes menores.
- Criar pull requests pequenos para facilitar revisao.
- Registrar criterios de aceite em cada issue do GitHub Projects.

## Stop

- Evitar concentrar muitas funcionalidades em uma unica pessoa.
- Evitar commits grandes misturando backend, testes e documentacao.
- Evitar implementar itens fora do escopo da Sprint 1 antes de fechar a base.

## Continue

- Manter backend separado em camadas.
- Usar migrations para mudancas de banco.
- Escrever testes para regras de permissao e autenticacao.
- Atualizar a documentacao junto com mudancas relevantes.

## Acoes concretas

- Cada membro deve assumir pelo menos uma issue da Sprint 1.
- Antes da review, rodar testes e demonstrar os endpoints.
- Registrar problemas encontrados no README ou em issues.
